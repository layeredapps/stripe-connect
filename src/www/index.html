Build web apps faster with Dashboard

  • User makes request
    • Dashboard receives request
      • URL recognized as Dashboard or Module content
        • User receives content from Dashboard server
      • Dashboard forwards request to your application server
        • Your application server returns content to Dashboard
          • User receives content from your application server

Dashboard provides your web application with an entire user and administrative interface, completely API-driven, and open source.

It has optional modules for Stripe integrations, OAuth and more, everything a web application needs. The server is stateless and scales horizontally by running multiple instances of your Dashboard server.

The Stripe Connect module adds a complete custom Stripe Connect integration for marketplace/platform payouts to your users, ready for you to integrate with your application.

Layered Apps

You can style Dashboard content by having your application serve a /public/template-additional.css with header and navigation styling, or /public/content-additional.css for content and form styling.

You can replace this page when you have a "/" ready on your application server:

const dashboard = require('@layeredapps/dashboard')
dashboard.start(__dirname)

You can use NodeJS for caching if you only have one Dashboard server, or Redis as a shared-cache if you have multiple instances of Dashboard.

export CACHE=redis|node
export REDIS_URL=redis://localhost:6379 
node main.js

Dashboard supports five different database platforms, PostgreSQL, MySQL, MariaDB, Microsoft SQL Server, IBM DB2 and if you're developing or understand the ramifications you can use SQLITE too! Just install the required driver and set the environment to use your preferred backend:

npm install mysql2 # mysql and mariadb
npm install pg pg-hstore # postgresql
npm install sqlite3
npm install tedious # mssql
npm install ibm_db
